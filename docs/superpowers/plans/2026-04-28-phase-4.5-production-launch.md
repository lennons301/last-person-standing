# Phase 4.5 — Production Launch Foundation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Take last-person-standing from "merges dormant to `main`" to "live on Vercel with CI/CD, secrets, and the cron pipeline running" before WC kickoff (2026-06-11).

**Architecture:** Mirrors moontide's 2026-04-14 infrastructure pattern: single Vercel project, Doppler-Vercel integration auto-syncs `prd` → Production env and `stg` → Preview env. Adapted for our extra dependencies (separate Upstash QStash projects per env, GitHub Actions cron, real Postgres in CI). Single PR/branch (`feature/phase-4.5-production-launch`) with three internal blocks: Provision → Deploy → Verify.

**Tech Stack:** Vercel (lhr1), Doppler, Neon (eu-west-2 London), Upstash QStash, GitHub Actions, Drizzle.

**Spec:** `docs/superpowers/specs/2026-04-28-phase-4.5-production-launch-design.md` — read this first if anything in the plan seems underspecified.

**Branch:** `feature/phase-4.5-production-launch` (already created).

### Task 8: Add the `migrate.yml` GitHub Action

**Files:**

- [ ] **Step 1: Create the workflow**

Contents:

```yaml
name: Migrate prod DB

on:
  workflow_run:
    workflows: [CI]
    branches: [main]
    types: [completed]

jobs:
  migrate:
    if: ${{ github.event.workflow_run.conclusion == 'success' }}
    runs-on: ubuntu-latest
    env:
      DATABASE_URL: ${{ secrets.PROD_DATABASE_URL }}
    steps:
      - uses: actions/checkout@v4
        with:
          ref: ${{ github.event.workflow_run.head_sha }}
      - uses: pnpm/action-setup@v4
        with:
          version: 9
      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: pnpm
      - run: pnpm install --frozen-lockfile
      - run: pnpm exec drizzle-kit migrate
```
